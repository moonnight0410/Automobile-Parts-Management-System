# 故障处理进度查询

## 概述

故障报告提交后，系统会生成唯一的故障ID，用户可以通过故障ID实时查询处理进度。所有处理记录都会上链存证，确保透明可追溯。

## 查询方式

### 方式1：通过故障报告列表查询

1. 登录系统
2. 进入"售后管理 > 故障报告"页面
3. 在故障报告列表中查看所有上报的故障
4. 列表会显示每个故障的状态和处理进度

### 方式2：通过故障ID精确查询

1. 登录系统
2. 进入"售后管理 > 故障报告"页面
3. 在页面顶部的搜索框中输入故障ID
4. 点击"查询"按钮
5. 系统会显示该故障的详细信息

### 方式3：通过AI助手查询

1. 进入"售后管理 > 智能售后助手"页面
2. 在输入框中输入问题，如："故障ID: FAULT-20250101-001的进度"
3. AI助手会识别意图并查询区块链数据
4. 返回当前的处理状态和进度

## 故障状态说明

### 1. 待审核（Pending）
- **含义**：故障报告已提交，等待客服审核
- **处理时间**：1-2个工作日
- **用户操作**：无需操作，等待审核结果
- **可能结果**：
  - 通过：进入下一步处理
  - 驳回：需要补充材料或重新上报

### 2. 审核通过（Approved）
- **含义**：故障报告审核通过，开始技术分析
- **处理时间**：2-3个工作日
- **用户操作**：无需操作
- **下一步**：技术团队分析故障原因

### 3. 技术分析中（Analyzing）
- **含义**：技术团队正在分析故障原因和影响
- **处理时间**：3-5个工作日
- **用户操作**：可能需要补充信息
- **分析内容**：
  - 故障原因分析
  - 影响范围评估
  - 处理方案制定

### 4. 等待处理（Waiting）
- **含义**：分析完成，等待安排处理
- **处理时间**：5-7个工作日
- **用户操作**：等待通知
- **处理方式**：
  - 维修更换
  - 召回处理
  - 其他补偿措施

### 5. 处理中（Processing）
- **含义**：正在执行处理方案
- **处理时间**：根据方案而定
- **用户操作**：配合处理（如到店维修）
- **处理类型**：
  - 维修：到授权维修中心
  - 更换：更换新零部件
  - 召回：批量召回处理

### 6. 已完成（Completed）
- **含义**：故障处理完成
- **用户操作**：确认处理结果
- **后续操作**：
  - 验收处理效果
  - 提交反馈意见
  - 查看处理记录

### 7. 已驳回（Rejected）
- **含义**：故障报告被驳回
- **驳回原因**：
  - 信息不完整
  - 不在质保范围内
  - 重复上报
  - 材料不真实
- **用户操作**：
  - 查看驳回原因
  - 补充材料后重新上报
  - 联系客服申诉

### 8. 已关闭（Closed）
- **含义**：故障处理完成且用户已确认
- **用户操作**：无需操作
- **后续**：如有新问题，需重新上报

## 处理进度时间线

### 标准处理流程

```
第1天：提交故障报告
  ↓
第2-3天：客服审核
  ↓
第4-6天：技术分析
  ↓
第7-10天：制定处理方案
  ↓
第11-15天：执行处理（维修/更换）
  ↓
第16天：完成并确认
```

### 加急处理流程

对于严重或危急故障，处理时间会缩短：

```
第1天：提交故障报告
  ↓
当天：客服审核（2小时内）
  ↓
第2天：技术分析（24小时内）
  ↓
第3天：制定方案（24小时内）
  ↓
第4-5天：执行处理
  ↓
第6天：完成并确认
```

## 查询结果说明

### 基本信息
- **故障ID**：唯一标识符
- **上报时间**：故障报告提交时间
- **当前状态**：故障处理当前阶段
- **处理进度**：百分比显示（0-100%）

### 处理记录
- **时间线**：按时间顺序显示所有处理步骤
- **操作人**：每个步骤的处理人员
- **操作内容**：每个步骤的具体操作
- **备注**：相关说明和注意事项

### 处理方案
- **方案类型**：维修、更换、召回等
- **执行时间**：方案开始执行的时间
- **预计完成**：预计完成的时间
- **负责机构**：执行方案的机构

### 相关信息
- **零部件信息**：零部件ID、名称、批次号
- **车辆信息**：VIN码、车型、车龄
- **质保状态**：是否在质保期内

## 常见问题

### Q1：故障ID忘记了怎么办？

A：可以通过以下方式找回故障ID：
1. 登录系统，在故障报告列表中查看
2. 查看提交成功的短信或邮件通知
3. 联系客服，提供上报时间和零部件信息查询

### Q2：处理进度长时间没有更新怎么办？

A：如果进度超过预期时间没有更新：
1. 检查故障状态是否为"处理中"
2. 联系客服了解具体情况
3. 如需补充材料，及时提供
4. 保留故障ID，便于后续查询

### Q3：可以催促处理进度吗？

A：可以。如果处理时间超过预期：
1. 通过故障ID查询进度
2. 点击"催促"按钮（如有）
3. 或联系客服说明情况
4. 客服会优先处理您的请求

### Q4：处理进度可以分享给他人吗？

A：可以。您可以：
1. 截图分享处理进度页面
2. 提供故障ID让对方自行查询
3. 联系客服，授权他人查询

### Q5：处理结果不满意怎么办？

A：如果对处理结果不满意：
1. 在系统中提交反馈意见
2. 详细说明不满意的原因
3. 提供相关证据材料
4. 客服会重新评估并处理

### Q6：故障处理需要多长时间？

A：处理时间根据故障严重程度而定：
- 轻微故障：7-10个工作日
- 一般故障：10-15个工作日
- 严重故障：5-7个工作日（加急）
- 危急故障：3-5个工作日（紧急）

### Q7：处理过程中可以补充材料吗？

A：可以。如果需要补充材料：
1. 客服会主动联系您
2. 也可以主动联系客服说明
3. 通过故障ID补充上传材料
4. 补充后处理会继续进行

### Q8：处理完成后还需要做什么？

A：处理完成后建议：
1. 到授权维修中心验收处理效果
2. 在系统中确认处理结果
3. 提交满意度评价和反馈
4. 保留处理记录，以备后续参考
